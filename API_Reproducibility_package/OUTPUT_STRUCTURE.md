
# Output Structure Documentation

This document explains how the evaluation script generates output files from the prompt dataset.

The goal is to preserve a transparent mapping between each input prompt and the responses generated by each large language model (LLM).

---

## Input Structure

The prompt dataset is stored in:

01_question_sets/

The directory contains four groups of prompt files:

Group1_molecules/
Group2_drugs/
Group3_comparisons/
Group4_inference/

Each prompt is stored as a Microsoft Word file (.docx).

Example input file:

alanine.docx

---

## Output Generation Rule

For every prompt file, the evaluation script generates **four separate response files**, one for each LLM.

Example:

Input prompt:

alanine.docx

Generated outputs:

alanine_ChatGPT.docx
alanine_Gemini.docx
alanine_Copilot.docx
alanine_Perplexity.docx

---

## Output Directory

All generated files are stored in a separate directory defined in the configuration:

OUTPUT_ROOT

The script preserves the same group structure as the input dataset.

Example:

02_api_answers/

Group1_molecules/
    alanine_ChatGPT.docx
    alanine_Gemini.docx
    alanine_Copilot.docx
    alanine_Perplexity.docx

Group2_drugs/
    ibuprofen_ChatGPT.docx
    ibuprofen_Gemini.docx
    ibuprofen_Copilot.docx
    ibuprofen_Perplexity.docx

Group3_comparisons/
    ketamine_ChatGPT.docx
    ketamine_Gemini.docx
    ketamine_Copilot.docx
    ketamine_Perplexity.docx

Group4_inference/
    warfarin_ChatGPT.docx
    warfarin_Gemini.docx
    warfarin_Copilot.docx
    warfarin_Perplexity.docx

---

## Content of Each Output File

Each generated Word file contains:

1. The original prompt text
2. The raw response returned by the model API
3. Metadata including:
   - model name
   - execution timestamp

The responses are stored exactly as returned by the API in order to preserve **raw output integrity**.

---

## Purpose

This structure ensures that:

- Each prompt can be traced to its model responses.
- Model outputs remain separated by provider.
- Researchers can easily compare model responses across platforms.

The one-to-one mapping between prompt files and response files also simplifies downstream analysis and verification.
